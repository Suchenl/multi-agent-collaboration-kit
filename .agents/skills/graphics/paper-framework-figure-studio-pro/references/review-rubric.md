# Review Rubric v3.2.4

Evaluate S2/S5 outputs against:

- paper fidelity and source-grounded terminology;
- required module coverage and visible core innovation mechanism;
- correct data/control/causal arrow semantics;
- precise color, line-style, symbol, and icon semantics;
- non-droppable core substep coverage;
- readable density, repetition compression, and redundancy control;
- figure-caption symbiosis and style-aware caption fit;
- first-glance appeal and 10-second comprehension;
- single primary reader path with 3-5 visual anchors;
- avoidance of poster/PPT-slide content dumping.

## S2/S5 Issue-Ledger Audit

S2 and S5 audits must not default to PASS/PASS_WITH_RISK/FAIL/BLOCKED-style candidate conclusions. They should produce an issue ledger per candidate:

- issue id;
- severity: minor, moderate, major, or blocker;
- category: source fidelity, core visibility, connector/edge, symbol/text, layout/density, caption alignment, caption burden, forbidden assumption, or style/stage fidelity;
- evidence basis;
- rerun-eligibility: easy, moderate, hard, not rerunable;
- must_fix_before_final;
- downstream_use: can use as primary direction, can absorb elements, needs rerun if selected, avoid as final basis;
- caption fields: alignment issues, burden, safe caption-only items, must-be-visible-not-caption-only items, legend needs, caption risk notes.

After S5, human decisions are outside this assistant workflow.

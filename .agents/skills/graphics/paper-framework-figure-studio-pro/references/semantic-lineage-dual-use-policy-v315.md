# Semantic Lineage And Dual-Use Artifact Policy v3.1.9

When a paper artifact, representation, generated pool, retrieved set, prototype bank, score, weight, validation proxy, reward, memory, or intermediate output feeds multiple downstream paths, record an artifact lineage table in S0/S1/S4 and carry it into S3 review and S4 prompt checks.

For each dual-use artifact, specify whether downstream uses are:

- same instance;
- sampled subset;
- same source pool;
- same distribution;
- regenerated batch;
- conceptual proxy;
- source-defined relation.

A figure must not imply raw sharing, exact-instance reuse, evaluation leakage, coordination, access, deployment, or causal direction that the paper does not support.

After S5, human decisions are outside this assistant workflow.

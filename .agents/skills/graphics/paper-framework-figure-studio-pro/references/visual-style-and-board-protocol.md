# Visual Style And Board Protocol

S2 may be exploratory and visually varied. S5 should be formal, paper-grounded, and clean.

Startup style choice is two-level. The saved F1-F4/atlas boards are first-level entry references for figure role, layout grammar, reader/detail pressure, and visual communication surface. They must be mapped into a second-level `style_lens_id` before S2 or S5 image generation. The second-level lens records paper narrative task, structure grammar, density budget, caption burden, icon/arrow/legend semantics, transfer boundary, and layer extraction/vector reconstruction risk.

Atlas boards and style references are not target-paper evidence. They may guide layout skeleton, reader path, density discipline, icon style, arrow grammar, callout strategy, and legend design, but must not import paper-specific facts, labels, modules, datasets, metrics, formulas, examples, claims, or exact figure structures.

For S1/S2 first-round sketches, style dimensions should be used to create paper-serving orthogonality, not decorative variety. The 8-sketch batch should vary useful dimensions such as layout grammar, topology/reader path, density, detail carrier, visual rhetoric, panel rhythm, caption burden, icon/arrow grammar, reviewer attraction hook, and aesthetic quality plan only when those variations help show the paper work or clarify the framework.

Second-round/formal candidate style:

- still generate raster images;
- use regular publication schematic composition;
- use stable modular geometry, clear connectors, restrained meaningful color, short labels, and paper-relevant icons;
- pair the visual style with a style-aware caption plan.

Avoid default hand-drawn, whiteboard, sketch-note, comic, sticky-note, painterly, photorealistic, texture-heavy, poster-like, or dense PPT-slide aesthetics for S5.

SVG/PPT editability is a secondary output-design consideration, not an output-modality change and not the primary figure objective. S5 candidates and the user-selected S5 final reference must be generated raster images.

After S5, human decisions are outside this assistant workflow.

Default first-glance budget: one reviewer takeaway, one 3-5 anchor primary reader path, one dominant main-flow region, and a compact support system. Avoid treating background context, caveats, legends, peer/entity groups, training details, data distributions, and update loops as equal-weight regions. If the candidate needs many true details, collapse repeated visual primitives or move them to caption instead of creating a crowded all-in-one map.
